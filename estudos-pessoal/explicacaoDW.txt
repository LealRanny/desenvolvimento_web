-Div: é um container, usa-se para envolver partes do código que devem ser tratadas como um bloco único. Facilita a aplicação de css.
	DIV E SECTION:
"Esse bloco de conteúdo precisa de um título?"
   ->Section(normalmente tem título): quando o conteúdo de dentro é de um conteúdo específico do site(ex: galeria de foto, sobre a empresa...). Section é para estrutura
   ->Div(normalmente fica dentro de section): serve para organizar layout(ex: envolver um ícone e um texto para ficarem lado a lado. Div é para estética(CSS) 

Você sabe que o esqueleto HTML está "pronto" quando você consegue ler o arquivo do topo ao fundo e encontrar todos os dados e informações que o exercício pediu, mesmo que a aparência esteja horrível.
   *HTML: Levantar as paredes, colocar os canos, a fiação e as portas. (A estrutura está lá).
   *CSS: Pintar, colocar o piso, escolher os móveis e a iluminação. (A estética).

	MEU CHECKLIST PRA PARTIR PRO CSS:
1. O conteúdo está todo lá? No meu exemplo, o exercício pediu 3 cartões com imagem, título, texto e botão. Você já colocou os três? Sim.
2. A hierarquia faz sentido? Se você desligasse todo o estilo do mundo, uma pessoa ainda conseguiria entender a ordem das informações? (Ex: O título <h3> vem antes do parágrafo <p>). No seu código, sim.
3. Eu dei "nomes" aos bois (Classes)? Para o CSS saber quem ele deve pintar de azul ou centralizar, você precisa de classes. Como eu já criei a <section class="cartoes"> e a <div class="card">, já dei os endereços para o CSS trabalhar.

	FLEXBOX:
-> Repara se as informações de cabeçalho estão uma em cada canto ou em partes diferentes, se estiver, é flexbox.(Não só no cabeçalho!)
Repare que na imagem dela, o nome "FlexSite" está de um lado e os links estão do outro.
Para conseguir esse efeito, o seu HTML precisa ter o <h1> e o <nav> como "irmãos" dentro do pai <header>.

	PARA BARRAS LATERAIS:
Onde vai esse Flexbox?
   O Flexbox que controla essa divisão lateral não vai no main, mas sim em um contêiner que abraça tanto a barra lateral quanto o conteúdo. Geralmente, fazemos isso logo abaixo do header.

Aqui está a lógica da estrutura:
	1. O "Pai de todos" (Contêiner): Uma <div> ou o próprio <body>.
	2. Filho 1 (Sidebar): Uma tag <aside> para o menu lateral.
	3. Filho 2 (Conteúdo): A sua tag <main> com os cartões.
Ao colocar display: flex(no CSS) nesse "Pai", o menu lateral e o main ficarão um ao lado do outro.

Ás vezes a página pode não estar mudando por causa disso:
   -> Mesma Pasta: O arquivo index.html e o style.css precisam estar "morando" na mesma pasta. Se o CSS estiver dentro de uma pasta chamada css, o caminho teria que ser href="css/style.css".
   -> Nome do Arquivo: Verifique se o arquivo se chama exatamente style.css (sem o "s" no final, tudo minúsculo). Se no HTML estiver style.css e o arquivo for estilo.css, não funciona.
