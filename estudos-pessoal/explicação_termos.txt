HTML:

<!DOCTYPE html>  -> declaração que avisa ao navegador que o documento é html.
<html lang="pt-br">  -> avisa que o conteúdo principal está em português
<meta charset="UTF-8">   -> diz ao navegador para usar tabelas de caracteres que inclui acentos e cedilha
<title>Kali Uchis Discography</title>     -> nome que vai aparecer na aba do navegador
<h1>  -> para título de cabeçalho. Só deve existir um <h1> por página.
<h2>  -> para subtítulos.
<a href="https..." target="_blank">Álbum</a>  -> acesso vai ser por o nome "Álbum" que aparecera ao lado da img
<a href="https..." target="_blank">    -> acesso será feito clicando na imagem





CSS:

	VELHO. AAAAAAAAA A MUDANÇA DA FLUIDEZ EM:
img {
                border-radius: 10px;
                transition: transform 0.3sease;   -> o correto é 0.3s ease. QUANDO ESTA ESCRITO CORRETO A FLUIDEZ É ABSURDAAAA
            }



display: flex;   -> transforma o elemento em um chefe de fila. É tipo um "a partir de agora, seus filhos devem ficar todos na mesma linha"
flex-wrap: wrap;   -> por padrão o flex espreme tudo na mesma linha. O wrap permite "quebrar". Se o espaço acabar, pule para a linha de baixo. É ISSO QUE PERMITE QUE O SITE FUNCIONE BEM VISUALMENTE NO CELULAR E COMPUTADOR.
justify-content: center;   -> alinha os itens no eixo horizontal. O center garante q se tiver só 2 albuns na última linha, eles fiquem centralizados.
gap: 30px;   -> serve para dar espaços entre os elementos sem precisar de cálculos. É o vão.
box-shadow: 0 4px 15px rgba(0,0,0,0,1);   -> para criar sombra. Significado dos números(deslocamento horizontal, deslocamento vertical, nível de desfoque, cor sombra). Nesse meu caso o rgba(0,0,0,0,1) é um preto bem transparente, é o que vai dar a elevação mais fluida.
border-radius: 20px;   -> define o quão redondas são as quinas.
font-family: ....;   -> é uma lista de fontes. Se o computador não tiver a primeira, ele tenta até chegar na última da lista.
font-size: 3em;   -> medida relativa, "3 vezes o tamanho padrão da letra". Usar assim é melhor do que usar "px", porque se adapta melhor a diferentes telas.
transition: transform 0.3s;   -> faz um movimento ser feito devagar e não abruptamente.

OBSERVAÇÃO:
	Em css usa ponto para a classe, em html não.
	Com ponto vc dá ordem apenas para quem tiver aquele apelido específico e o que estiver dentro dele -> ex: class = "galeria" -> .galeria
	Sem ponto vc dá ordem para a tag HTML, e modifica todas as seções do site.

Para trazer movimento para um item quando passar o mouse em cima dele:
	:hover  -> significa pairar. É adicionado a frente do nome do elemento.
	transform: scale()  -> é aqui que indica quantos porcento a imagem vai aumentar quando passar o mouse em cima.
